## Términos que NO se traducen

Mantener en inglés en ambos idiomas:
- Git, GitHub, GitLab, Bitbucket
- JavaScript, Python, Java, Go, Rust, etc.
- React, Angular, Vue, Svelte
- Node.js, Django, Rails, Spring
- AWS, GCP, Azure
- Linux, Windows, macOS
- HTTP, HTTPS, TCP/IP, DNS
- JSON, XML, YAML, CSV
- SQL, PostgreSQL, MySQL, MongoDB, Redis
- npm, yarn, pip, cargo
- CLI, GUI, IDE
- API, SDK, CLI, GUI
- SaaS, PaaS, IaaS
- SLI, SLO, SLA

---

## Convenciones de Traducción

1. **Términos técnicos:** Preferir mantener en inglés si es estándar de la industria
2. **Roles:** Traducir descriptivamente pero mantener siglas (DBA, QA)
3. **Marcas comerciales:** Nunca traducir (GitHub Copilot, AWS)
4. **Siglas:** Mantener en mayúsculas (CI/CD, TDD, LLM)
5. **Verbos de acción:** Usar imperativo en ambos idiomas ("Analiza" / "Analyze")
