# 11-10 — Plantilla de runbook operativo

```text
Objetivo:
Estandarizar la ejecución de tareas críticas o frecuentes.

Inputs requeridos:
- tarea: [NOMBRE DE LA TAREA]
- responsable: [NOMBRE]
- frecuencia: [DIARIA / SEMANAL / AD-HOC]
- sistemas afectados: [LISTA]

Pasos:
1. [PASO 1: DESCRIPCIÓN]
2. [PASO 2: DESCRIPCIÓN]
3. [PASO 3: DESCRIPCIÓN]
...
N. Validar resultado y documentar evidencia.

Checklist de validación:
- [ ] Todos los pasos ejecutados.
- [ ] No se detectaron errores.
- [ ] Se notificó a los interesados.
- [ ] Evidencia adjunta.

Campos personalizables:
- [TAREA], [RESPONSABLE], [FRECUENCIA], [SISTEMAS]
```

## Descripción

Plantilla para documentar procedimientos paso a paso. Facilita la ejecución controlada y la transferencia de conocimiento operativo.

---

Ejemplo de uso:

- tarea: Reinicio de base de datos
- responsable: Laura Méndez
- frecuencia: AD-HOC
- sistemas afectados: PostgreSQL
